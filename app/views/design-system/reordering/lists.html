Re-ordering

Lists

How it works

The basic flow is:

  1. Users configure a question page with a list of options, such as, radios or checkboxes.
  2. If the list has more than one option, they can select the Edit List button to manage the options.
  3. In the edit list view, users can:
    • Reorder items using drag-and-drop or "Up" and "Down" buttons.
    • Remove items from the list.
    • Save changes to return to the question configuration page.
  4. Reordering is supported through:
    • Dragging and dropping list items to update their order.
    • "Up" and "Down" buttons for precise movement.

Changes are only applied after selecting Save Changes, ensuring the main configuration page reflects the updates.

Rules and actions

  • The Edit List button is only visible if more than one list item exists.
  • First and last list items have restricted CTAs:
    • First item: "Down" button only.
    • Last item: "Up" button only.
  • Items in between have both "Up" and "Down" buttons.
  • Every list item has a "Remove" button to delete it from the list.
  • Users must confirm their changes by selecting Save Changes before returning to the main page.

This flow ensures users can manage their list options effectively while maintaining clarity and control over their configuration.

The Highlight on Focus feature enhances user interaction by visually indicating which list option is being interacted with.
