What is the overall flow?

Below is a visual illustration of the general flow. You can read more details about this flow on the design system user journey tab .

Types of Pages

Form management pages

Listing page

A summary of all the pages the user has created, including 'End pages' such as the Check Answers page. When JavaScript is enabled, users can access a quick preview of each page.

  • Pages are represented using summary cards.
Re-ordering page

Change the order of pages by moving them up or down.

Preview form

Preview the form from start to finish.

Form creation pages

Page type selection

Choose to create a question or a guidance page.

Information type selection

Choose the type of question to add.

Question configuration

Configure a question that appears on a page.

Page overview

View a summary of questions, add headings and guidance, allow multiple responses, and re-order questions.

Progressive Enhancement

The user journeys for creating pages follow progressive enhancement principles, ensuring a quality service for all users, regardless of whether JavaScript is enabled. The experience may vary slightly for users without JavaScript, but the outcomes remain consistent and accessible.

Comparison: JavaScript enabled vs. disabled (see also a Figma overview of visual differences)
Feature When JavaScript is disabled When JavaScript is enabled
Preview panels Preview panels are not shown together with configuration panels. Preview panels are shown alongside configuration panels.
Preview buttons
  • Open a preview of the question/page being created
  • View the error messages for a question
  • Preview buttons are removed and replaced by skip links that are only visible when Tabbing activates it. (See overview of this in Figma kit of parts)
  • Skip links jump users from the configuration panel to a preview tab (Page preview or Error messages)
  • Skip links inside preview tabs return users to the configuration panel
Preview panel styling No additional styling is applied to questions in the preview panel. Questions in the preview panel receive a left-border.
Guided input No guided input or content highlighting in the preview area. When users focus on an input field, the related content in the preview area gets highlighted.
Re-ordering movements Page reloads are required after each re-ordering action.
  • Highlight on focus: When focus is applied to an input field or button, a highlight appears on the corresponding preview element. The highlight is removed when the focus moves away.
  • Types of reordering:
    • Lists (checkboxes, radios) – see Figma and the design system
    • Pages – see Figma and the design system
    • Questions – see Figma and the design system
  • Specific screen reader announcements must be applied for accessibility.
